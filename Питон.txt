Это самый правильный и безопасный способ. Вы создаете изолированную среду для своего проекта, где у вас будет полный контроль над версиями пакетов и самого pip.

    Установите необходимый пакет для создания окружений (если еще не установлен):
    bash

    sudo apt install python3-venv python3-full -y

    Создайте виртуальное окружение, например, с именем myenv:
    bash

    python3 -m venv myenv

    Активируйте его:
    bash

    source myenv/bin/activate

    После активации в начале строки терминала появится (myenv).

    Обновите pip внутри окружения (теперь это безопасно):
    bash

    pip install --upgrade pip

    Обратите внимание: здесь не нужен sudo и используется просто pip, а не pip3 (хотя pip3 тоже сработает).

    Устанавливайте любые пакеты (например, requests):
    bash

    pip install requests

Чтобы выйти из окружения, используйте команду:
bash

deactivate

pip install pyinstaller
